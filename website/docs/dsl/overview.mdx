# DSL Overview (`version = "0.5"`)

Use this page for DSL shape. Use [DSL Reference Table](./reference-table) for factual field constraints.

Core shape:

```bash
version = "0.5"
alias b = build

build [target] [mode="release"]: deps...
    @in ...
    @out ...
    @env KEY=value
    @secret_env TOKEN
    @dir apps/api
    @requires cargo
    @mode graph
    @isolation off
    run command
```

## Execution inference rules

- task with declared `@out` defaults to graph behavior
- task without `@out` defaults to interactive behavior
- explicit `@mode` can override inference

## Minimal high-signal patterns

### Graph task pattern

```bash
build:
    @in src/**/* Cargo.toml
    @out target/release
    cargo build --release
```

### Interactive task pattern

```bash
dev:
    @mode interactive
    npm run dev
```
